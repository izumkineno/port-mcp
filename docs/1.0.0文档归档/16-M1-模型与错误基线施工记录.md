# 16 M1 模型与错误基线施工记录

本文档承接 [03-初版工具契约](03-初版工具契约.md)、[05-状态机与资源生命周期](05-状态机与资源生命周期.md)、[06-并发与缓冲区设计](06-并发与缓冲区设计.md)、[07-错误模型与返回格式](07-错误模型与返回格式.md)、[08-测试与验收计划](08-测试与验收计划.md)、[10-开发里程碑](10-开发里程碑.md)、[13-代码施工大纲](13-代码施工大纲.md) 与 [15-M0-工程骨架施工记录](15-M0-工程骨架施工记录.md)，用于记录 M1 阶段的模型、统一返回、错误、ID、时间、脱敏、数据摘要和单元测试证据。

本文档是 M1 施工记录。M1 已完成模型、统一返回、错误、ID、时间、脱敏、payload 摘要、限制值和单元测试证据；不得据此开始 M2 RuntimeRegistry、资源锁、队列、缓冲区或真实 I/O 实现，除非另行按 17 的 M2 plan 授权。

## 记录结论

当前 M1 已完成。M1 没有实现实例表或真实端口通信，而是固定跨层共享模型与错误返回基线，让后续 `app`、`runtime`、`transport` 和 `mcp` 层依赖同一套稳定数据结构。

| 项目 | 当前状态 | 说明 |
| --- | --- | --- |
| 实例类型与状态模型 | 已完成 | `InstanceType` 覆盖 `Serial`、`TCP`、`UDP`；`InstanceState` 覆盖七种状态；`Released` 明确为非持久状态。 |
| 配置模型 | 已完成 | `SerialConfig`、`TcpConfig`、`UdpConfig`、`ConfigSnapshot`、默认值和基础校验已落地。 |
| 统一成功/失败返回 | 已完成 | `ToolResponse`、`SuccessResponse`、`FailureResponse` 覆盖 07 顶层字段；失败不包含 `warnings`。 |
| 错误 category/code | 已完成 | `ErrorCategory`、`ErrorCode`、`DomainError` 覆盖 M1 主要 category/code 和 details。 |
| ID 与时间 | 已完成 | `HandleId`、`RequestId`、`ErrorId`、`Timestamp`、`IdGenerator` 已落地，timestamp 使用 UTC/RFC3339。 |
| 脱敏与 details 裁剪 | 已完成 | `Redactor`、`ErrorDetails` 覆盖本地路径、环境变量、payload、OS 错误和堆栈脱敏。 |
| 数据摘要与编码 | 已完成 | `Payload`、`PayloadEncoding`、`PayloadSummary` 覆盖 text/hex、append line break、preview、截断和 datagram 标记。 |
| 单元测试证据 | 已完成 | `cargo test` 通过 9 个测试，其中 8 个为 M1 单元测试，1 个为 M0 smoke 测试。 |

结论：M1 模型与错误基线已完成；后续进入 M2 前，应继续保持 handler 不临时拼 JSON，外部库原始错误不得穿透到工具返回。

## M1 目标

M1 的目标是把初版工具契约中跨层共享的概念转成稳定、可序列化、可测试、可脱敏的模型基线。

M1 完成后应具备：

- 连接类型、实例状态、状态摘要、配置快照和统计摘要有稳定模型。
- 所有工具成功和失败返回都能由强类型结构序列化得到。
- 领域错误能表达 07 定义的 category、code、message、recovery_hint、retryable 和 details。
- 参数错误、状态错误、资源冲突、连接超时、读超时、写失败和容量限制能映射到稳定错误外形。
- `handle_id`、`request_id`、`error_id` 生成规则可测试、可读、进程内唯一。
- timestamp 使用统一 UTC/RFC3339 口径。
- 脱敏和 details 裁剪规则有单元测试，默认工具返回不暴露敏感信息。
- text/hex 编码、payload preview、截断和统计摘要有统一工具函数或模型表达。
- 单元测试覆盖序列化、错误模型、脱敏、ID、时间和基础配置校验。

## M1 不做事项

| 不做事项 | 原因 |
| --- | --- |
| 不实现 RuntimeRegistry、实例表或会话绑定表 | 属于 M2。M1 只定义模型和错误基线。 |
| 不实现资源锁、`closing` tombstone 或 stale lock 诊断 | 属于 M2。M1 只定义可返回的错误类别和 details 外形。 |
| 不实现发送队列、接收缓冲区、订阅者队列或预算扣账 | 属于 M4。M1 只定义限制值和统计字段模型。 |
| 不打开串口、TCP socket 或 UDP socket | 属于 M5/M6。M1 只定义配置和错误映射。 |
| 不实现初版工具业务 handler | M1 可提供返回包装辅助，但不应接入真实工具服务。 |
| 不把 MCP SDK 类型放入模型 | SDK 类型只允许封装在 `mcp` 层。 |
| 不让 `anyhow::Error`、`serialport::Error` 或 Tokio I/O 错误成为工具返回模型 | 外部错误必须先转换为项目领域错误。 |

## 能力域施工记录

### 1. 实例类型、状态与摘要

M1 必须固定初版可见实例类型和状态集合。

| 检查项 | 结果 | 证据 |
| --- | --- | --- |
| 实例类型覆盖 `Serial`、`TCP`、`UDP` | 通过 | `InstanceType::{Serial,Tcp,Udp}` 序列化为 `Serial`、`TCP`、`UDP`。 |
| 状态集合覆盖 03/05 定义的七种状态 | 通过 | `InstanceState` 覆盖 `Created/Configured/Connected/Disconnecting/Disconnected/Error/Released`。 |
| `Released` 不作为实例表长期状态保存的语义已表达 | 通过 | `InstanceState::Released.is_persistent()` 返回 `false`。 |
| 状态摘要能表达 `handle_id`、`type`、`state`、配置摘要、统计和 `last_error` | 通过 | `InstanceSummary` 包含 `handle_id`、`type`、`state`、`resource`、`config`、`stats`、`last_error`。 |
| 状态序列化名称与文档大小写一致 | 通过 | `unit_state_model_serializes_documented_names_and_summary_shape` 验证。 |

验收要求：状态模型不得引入 03/05 未记录的新状态；若实现需要内部中间状态，必须只作为内部状态，不进入工具返回外形。

### 2. 配置模型与参数校验

M1 应定义 serial、tcp、udp 配置输入和配置快照。配置校验失败必须能映射到 `InvalidArgument`，并携带可定位的 `details.field`。

| 能力 | M1 记录项 | 结果 |
| --- | --- | --- |
| Serial 配置 | port、baudrate、data bits、stop bits、parity、flow control、timeout、encoding 等字段口径。 | 通过 |
| TCP 配置 | client/listen 模式、host/ip、port、timeout、connect 相关字段。 | 通过 |
| UDP 配置 | bind 地址、remote 地址、端口、timeout、datagram 语义相关字段。 | 通过 |
| 默认值 | 与 11 的运行配置和 06 的限制值保持一致。 | 通过 |
| 参数错误 | 缺字段、类型错误、非法枚举、越界、非法地址、类型不匹配。 | 通过 |

证据回填：

| 检查项 | 结果 | 证据 |
| --- | --- | --- |
| 参数缺失返回 `MISSING_REQUIRED_FIELD` | 通过 | `validate_required_field("port", None)`。 |
| 类型错误返回 `INVALID_FIELD_TYPE` | 通过 | `ErrorCode::InvalidFieldType` 已纳入 `unit_error_codes` 覆盖。 |
| 枚举非法返回 `INVALID_ENUM_VALUE` | 通过 | `ErrorCode::InvalidEnumValue` 已纳入 `unit_error_codes` 覆盖。 |
| 数值越界返回 `INVALID_RANGE` | 通过 | `validate_tcp_port(70000)` 与 `validate_force_close_deadline(0)`。 |
| 实例类型不匹配返回 `TYPE_MISMATCH` | 通过 | `validate_instance_type(Serial, Tcp)`。 |
| 非法地址返回 `INVALID_ADDRESS` | 通过 | `TcpConfig::client("0.0.0.0", 9000).validate_remote()`。 |

| `ResourceBusy` | `SERIAL_PORT_BUSY`、`TCP_LISTEN_ADDR_BUSY`、`UDP_BIND_ADDR_BUSY`、`RESOURCE_CLOSING`、`RESOURCE_LOCK_STALE` | 通过 | `unit_error_codes_cover_m1_required_categories_and_details`。 |

错误构造必须能表达：

- 当前错误是否可重试。
- 对实例状态的影响摘要。
- 面向 AI 的下一步恢复建议。
- 脱敏后的 `details`。
- 可关联日志的 `error_id` 或 `last_error_id`。

### 5. `error.details` 与脱敏

M1 必须先把可返回 details 的稳定键名和脱敏边界落地。

推荐 details 键：

| 键 | 用途 | 结果 |
| --- | --- | --- |
| `field`、`expected_type`、`actual_type`、`allowed_values` | 参数错误定位。 | 通过：`ErrorDetails` 支持稳定键。 |
| `min`、`max`、`actual`、`requested`、`limit` | 范围和容量错误。 | 通过：范围和容量校验写入 details。 |
| `current_state`、`allowed_states`、`next_actions` | 状态错误恢复。 | 通过：`ErrorDetails` 支持稳定键，M2 状态矩阵接入具体值。 |
| `resource`、`owner_handle_id` | 资源冲突或连接失败。 | 通过：`ResourceSummary` 与 `ErrorDetails` 支持稳定外形。 |
| `queue_items`、`buffer_bytes` | 队列和缓冲区错误。 | 通过：`ErrorDetails` 支持稳定键，M4 接入具体值。 |
| `task`、`io_kind` | 后台任务和 I/O 错误。 | 通过：`Redactor::os_error` 返回 `io_kind`。 |
| `last_error_id` | 实例故障关联。 | 通过：`LastErrorSummary` 与 `ErrorId` 已落地。 |

脱敏证据回填：

| 信息类型 | 工具返回策略 | 结果 | 证据 |
| --- | --- | --- | --- |
| 串口名 | 可返回安全摘要。 | 通过 | `ResourceSummary::serial("COM3")`。 |
| IP 和端口 | 可返回。 | 通过 | `ResourceSummary::tcp("127.0.0.1", 9000)`。 |
| 完整本地路径 | 默认不返回完整路径。 | 通过 | `Redactor::local_path` 只返回文件名。 |
| 系统用户名 | 不返回。 | 通过 | `unit_redaction` 验证 `alice` 不进入返回。 |
| 环境变量值 | 不返回。 | 通过 | `Redactor::env_value` 返回 `<redacted>`。 |
| 原始 payload | 只返回短 preview、编码和省略长度。 | 通过 | `PayloadSummary` 与 `Redactor::payload_preview`。 |
| OS 错误文本 | 返回脱敏摘要和 `io_kind`。 | 通过 | `Redactor::os_error` 返回 `permission_denied` 和 `<redacted>`。 |
| panic 或堆栈 | 不进入工具返回。 | 通过 | `Redactor::stack_trace` 返回 `<redacted>`。 |

### 6. ID、时间与日志关联

M1 必须让工具返回、日志和 `last_error` 能通过 ID 关联。

| 能力 | 要求 | 结果 | 证据 |
| --- | --- | --- | --- |
| `handle_id` | 进程内唯一，可读，能区分 serial/tcp/udp。 | 通过 | `IdGenerator::next_handle_id` 生成 `h_ser_001`、`h_udp_001`。 |
| `request_id` | 每次工具调用唯一，可关联日志。 | 通过 | `IdGenerator::next_request_id` 生成递增 `req_...`。 |
| `error_id` | 每个可记录错误唯一，可进入 `last_error` 或 details。 | 通过 | `IdGenerator::next_error_id` 与 `LastErrorSummary`。 |
| `timestamp` | UTC/RFC3339，成功和失败返回使用同一生成口径。 | 通过 | `Timestamp::now_utc` 与 `Timestamp::from_rfc3339`。 |
| 单调耗时 | 可用于日志 duration，不替代 timestamp。 | 通过 | M1 不实现日志计时器；文档语义保留，后续工具调用日志使用 duration 字段，不替代 `Timestamp`。 |

ID 生成器可以是自研递增或短随机方案。若引入 `uuid` 等新依赖，必须回写 [09-库选择与依赖决策](09-库选择与依赖决策.md)。

### 7. 数据编码、payload 摘要与统计字段

M1 不实现真实队列和缓冲区，但必须让后续 M3/M4 能共用同一套数据表达。

| 能力 | 要求 | 结果 | 证据 |
| --- | --- | --- | --- |
| text 编码 | 默认 UTF-8，编码失败映射到 `TEXT_ENCODING_FAILED`。 | 通过 | `Payload::from_text`；`TEXT_ENCODING_FAILED` 纳入 error code 覆盖。 |
| hex 编码 | 支持合法十六进制，非法输入映射到 `INVALID_HEX`。 | 通过 | `Payload::from_hex("70696e67")` 和非法 hex 测试。 |
| append line break | 能表达发送前追加换行的字节结果。 | 通过 | `Payload::from_text("ping", true)` 返回 `ping\n`。 |
| payload preview | 支持裁剪、编码标记和 `omitted_bytes`。 | 通过 | `PayloadSummary::from_bytes`。 |
| datagram 标记 | 能为 UDP 订阅或摘要表达 `datagram=true`。 | 通过 | `PayloadSummary` 包含 `datagram`。 |
| 统计字段 | 能表达 tx/rx 字节数、丢弃量、队列长度、订阅者数量、最近活动时间。 | 通过 | `InstanceStats` 覆盖统计字段。 |

### 8. 限制值与配置边界

M1 应定义默认限制和硬上限模型，让后续 M4 动态预算实现有稳定输入。

| 限制域 | 来源 | 结果 | 证据 |
| --- | --- | --- | --- |
| tx queue 和 tx frame | 06、11 | 通过 | `RuntimeLimits` 默认 `256`、`64 KiB`。 |
| rx buffer 和 pull 上限 | 06、11 | 通过 | `1 MiB`、`4 KiB`、`64 KiB`。 |
| subscriber queue、payload、数量和限速 | 06、11 | 通过 | `128`、`16 KiB`、`32`、`64/256/1024`、burst `128`。 |
| scan 并发、端口数、超时和 loopback 限制 | 07、08、11 | 通过 | `scan_max_concurrency=64`、`scan_max_ports=256`、`scan_total_timeout_ms=10000`、`scan_allowed_hosts=[127.0.0.0/8, ::1]`。 |
| `max_instances` | 06、11 | 通过 | 默认 `64`。 |
| `max_total_buffer_bytes` 与 `max_total_queued_bytes` | 06、11 | 通过 | 默认 `128 MiB`、`64 MiB`，含绝对硬上限常量。 |
| `force_close_deadline_ms` | 11 | 通过 | 默认 `5000`，校验范围 `1000..30000`。 |

限制值越界必须能映射到 `InvalidArgument/INVALID_RANGE` 或对应容量错误；不得在运行时深处才产生不可解释错误。
