# 01 端口 MCP 初版工程功能设计大纲

端口 MCP 面向串口、TCP/UDP 通信调试，目标是让 AI 能够稳定地创建连接实例、收发数据、观察接收流、查询状态，并把调试过程沉淀为可复用记录。本文档决定初版工程的功能设计大纲，供后续 AI 拆分任务、实现功能、编写测试和审查验收使用。

本文档只覆盖初版必须落地的能力。VISA 仪器控制、Modbus 等协议辅助、心跳、异常注入和高级流式触发策略拆分到 [02-进阶实现大纲](02-进阶实现大纲.md)。

## 初版目标

- 支持串口、TCP、UDP 三类基础连接。
- 支持多实例、多 AI 会话同时调用，连接实例之间互不干扰。
- 支持实例创建、配置、连接、断开、查询、切换和释放。
- 支持基础发送、拉取式读取、缓冲区清理。
- 支持最小流式订阅，让多个 AI 会话可以同时观察同一实例的接收数据。
- 明确资源占用、状态转移、并发读写、错误返回和验收标准。

## 初版非目标

- 不实现 VISA 仪器资源管理。
- 不内置 Modbus、SCPI、AT 指令等协议打包解析工具。
- 不实现心跳任务、异常注入、链路扰动模拟。
- 不支持复杂流式触发策略，例如正则过滤、十六进制组帧、时间/大小聚合。
- 不提供完整逐字段 JSON Schema 手册；工具参数和返回值只定义到足够指导实现的功能级别。

## 核心实体模型

端口 MCP 需要区分物理资源、连接实例和 AI 会话三个层级。

| 层级 | 示例 | 初版职责 |
| --- | --- | --- |
| 物理资源 | `COM1`、`192.168.1.10:502`、`0.0.0.0:9000/udp` | 表示真实串口或网络端点。串口默认独占；TCP/UDP 按实例配置创建 socket。 |
| 连接实例 | `h_ser_001`、`h_tcp_002`、`h_udp_003` | 保存连接类型、配置、状态、缓冲区、收发计数、订阅列表和后台任务句柄。 |
| AI 会话 | 当前 MCP Client 会话 | 可通过 `instance_use` 绑定默认实例；可独立订阅某个实例的接收流。 |

### 句柄生命周期

`handle_id` 是所有工具操作连接实例的稳定标识。初版建议按类型生成可读前缀：

- 串口：`h_ser_001`
- TCP：`h_tcp_001`
- UDP：`h_udp_001`

生命周期如下：

```text
Created -> Configured -> Connected -> Disconnecting -> Disconnected -> Released
              |              |
              |              -> Error
              -> Released
```

状态说明：

- `Created`：实例已创建，但尚未写入完整连接配置。
- `Configured`：实例已有可连接配置，但未占用底层资源。
- `Connected`：底层资源已打开，后台接收/发送任务已启动。
- `Disconnecting`：正在停止后台任务并释放底层资源。
- `Disconnected`：连接已关闭，配置仍保留，可再次连接。
- `Released`：实例从全局实例表移除，后续操作应返回句柄不存在。
- `Error`：连接或后台任务发生错误，需允许查询错误原因并断开释放。

## 初版工具矩阵

### 实例管理

| 工具名称 | 关键参数 | 返回结果 | 实现约束 |
| --- | --- | --- | --- |
| `instance_create` | `type` | 全局唯一 `handle_id` 与初始状态 | `type` 仅支持 `Serial`、`TCP`、`UDP`。创建后状态为 `Created`。 |
| `instance_list` | 无 | 当前所有连接实例摘要 | 只读操作，返回类型、状态、创建时间、收发计数和占用资源摘要。 |
| `instance_query` | `handle_id` | 指定实例详情 | 返回配置、状态、连接时长、缓冲区长度、订阅数量、最后错误。 |
| `instance_use` | `handle_id` | 当前 AI 会话绑定状态 | 仅改变当前 AI 会话默认实例，不改变连接实例本身。 |
| `instance_release` | `handle_id`, `force` | 释放成功状态 | 普通释放不自动断开 `Connected` 实例；应先 `port_disconnect`，或显式 `force=true` 进入强制释放流程。 |

### 连接配置

| 工具名称 | 关键参数 | 返回结果 | 实现约束 |
| --- | --- | --- | --- |
| `serial_config` | `handle_id`, `port`, `baudrate`, `bytesize`, `stopbits`, `parity`, `flow_control`, `timeout_ms` | 配置后的实例状态 | 只能配置 `Serial` 实例。串口参数需校验合法范围。 |
| `tcp_udp_config` | `handle_id`, `protocol`, `ip`, `port`, `mode`, `timeout_ms` | 配置后的实例状态 | `protocol` 为 `TCP` 或 `UDP`；`mode` 初版支持 `client` 和 `server/listen` 的最小可用语义。 |

串口初版参数约定：

- `bytesize`：`5`、`6`、`7`、`8`。
- `stopbits`：`1`、`1.5`、`2`。
- `parity`：`None`、`Odd`、`Even`。
- `flow_control`：`None`、`RTS-CTS`、`XON-XOFF`。
- `timeout_ms`：读写默认超时，必须有合理上限。

TCP/UDP 初版参数约定：

- `ip`：目标地址或监听地址。
- `port`：`1..65535`。
- `mode=client`：主动连接或指定远端。
- `mode=server/listen`：监听本地地址；TCP 初版固定为单客户端 listen 策略，接受第一个活动连接，后续连接按实现策略拒绝或立即关闭，并且策略与当前活动客户端必须可查询。

### 端口行为

| 工具名称 | 关键参数 | 返回结果 | 实现约束 |
| --- | --- | --- | --- |
| `port_scan` | `type`, `range` | 可用串口或网络端口探测结果 | 初版必须支持串口枚举；网络端口扫描默认只允许 loopback 目标，并限制端口数量、并发数、总超时和结果数量。 |
| `port_connect` | `handle_id` | `Connected` 状态与连接时间戳 | 只能连接 `Configured` 或 `Disconnected` 实例。成功后启动收发后台任务。 |
| `port_disconnect` | `handle_id` | `Disconnected` 状态 | 停止后台任务、释放底层资源、保留配置和统计。 |
| `port_send` | `handle_id`, `data`, `encoding`, `append_line_break` | 发送字节数与成功标识 | 发送进入实例发送队列，由后台任务串行写入。 |
| `port_pull` | `handle_id`, `timeout_ms`, `max_bytes` | 接收缓冲区中的数据 | 消耗式读取，读取后从普通接收缓冲区移除。 |
| `port_clear` | `handle_id`, `buffer_type` | 清空成功状态 | `buffer_type` 支持 `tx`、`rx`、`all`。 |

`data` 初版应至少支持两种编码：

- `text`：按指定文本编码转字节，默认 UTF-8。
- `hex`：空格分隔或连续十六进制字符串转字节。

### 最小流式订阅

| 工具名称 | 关键参数 | 返回结果 | 实现约束 |
| --- | --- | --- | --- |
| `port_subscribe_stream` | `handle_id`, `trigger_condition` | 订阅成功状态与 stream URI | 初版只要求 `on_receive`，即收到数据后广播给订阅会话。 |
| `port_unsubscribe_stream` | `handle_id` | 取消订阅状态 | 只取消当前 AI 会话在该实例上的订阅，不影响其他会话。 |

初版最小订阅语义：

- 接收任务读取到底层数据后，同时写入普通 `rx` 缓冲区和订阅广播通道。
- `port_pull` 仍然只消费普通 `rx` 缓冲区，不影响已广播给订阅者的数据。
- 多个 AI 会话订阅同一 `handle_id` 时，应各自收到同一份接收数据通知。
- 初版触发条件只支持 `on_receive`；正则、Hex 组帧、按行、按时间/大小合并进入进阶大纲。
- 为避免上下文爆炸，单次通知必须有大小上限，超过上限应截断或分片并在通知中标记。

可使用 MCP 资源更新通知表达数据到达：

```json
{
  "method": "notifications/resources/updated",
  "params": {
    "uri": "ports://h_ser_001/stream",
    "data": "41 42 43 44 0D 0A",
    "encoding": "hex",
    "truncated": false
  }
}
```

如果目标 MCP SDK 对通知能力有限，初版可以用事件队列模拟订阅，但文档和实现必须保留广播式语义。

## 并发与资源语义

### 写入并发

- 每个连接实例维护独立发送队列。
- 多个 AI 会话同时调用 `port_send` 时，请求按进入队列顺序串行写入底层资源。
- 单个 `port_send` 的数据帧不可被其他请求插入。
- 发送成功后更新实例发送字节数、发送帧计数和最后发送时间。

### 读取并发

- 后台接收任务是底层资源进入系统的唯一数据入口。
- 接收到的数据先进入实例普通 `rx` 缓冲区，再进入订阅广播流程。
- `port_pull` 是消耗式读取，适合一问一答协议。
- `port_subscribe_stream` 是广播式观察，适合多 AI 同时监听同一通道。

### 资源占用

- 串口物理资源默认独占。一个串口被某个 `Connected` 实例占用后，其他实例连接同一串口必须返回资源占用错误。
- TCP client 实例默认独立 socket，可对同一远端创建多个连接，除非底层系统限制。
- TCP listen/server 模式必须明确监听地址冲突：同一地址和端口不能被多个实例同时监听。
- UDP bind/listen 模式默认不共享同一地址和端口，除非实现显式支持端口复用并在状态中标出。

## 错误模型

所有工具失败时应返回结构化错误，至少包含错误类别、可读消息、关联 `handle_id` 和可恢复建议。

| 错误类别 | 典型场景 | 处理建议 |
| --- | --- | --- |
| `HandleNotFound` | 操作不存在或已释放的 `handle_id` | 调用 `instance_list` 获取当前实例。 |
| `InvalidState` | 未配置时连接、未连接时发送 | 查询实例状态并按生命周期补齐前置步骤。 |
| `InvalidArgument` | 串口参数非法、端口号越界、Hex 数据格式错误 | 修正参数后重试。 |
| `ResourceBusy` | 串口已被占用、监听端口冲突 | 换资源或断开占用实例。 |
| `ConnectTimeout` | TCP 连接超时、串口打开超时 | 检查设备、地址和超时设置。 |
| `ReadTimeout` | `port_pull` 在超时时间内无数据 | 可重试或改用订阅。 |
| `WriteFailed` | 底层写入失败或连接中断 | 查询实例最后错误并断开重连。 |
| `BufferLimitExceeded` | 发送队列满、单帧过大、拉取请求过大、订阅 payload 请求过大、扫描范围或结果超过上限 | 降低请求规模、分批处理、等待队列排空或缩小扫描范围。 |

## 状态管理要求

- 使用全局实例表保存 `handle_id -> instance` 映射。
- 每个实例至少维护：类型、状态、配置、创建时间、连接时间、最后活动时间、收发计数、普通 `rx` 缓冲区、发送队列、订阅列表、最后错误。
- AI 会话绑定关系应与连接实例分离，避免一个会话调用 `instance_use` 影响其他会话。
- 断开连接必须停止接收任务、发送任务和订阅广播入口，并释放底层资源。
- 释放实例必须清理缓冲区、队列、订阅关系和资源锁。

## 数据安全与限制

- `max_bytes` 必须设置默认值和最大值，避免一次 `port_pull` 返回过大数据。
- 普通 `rx` 缓冲区必须有容量上限；初版采用丢旧保新策略并累计丢弃统计，不把普通 `rx` 溢出作为工具失败返回。
- 流式通知必须有单次 payload 上限和频率保护。
- 网络端口扫描必须限制目标、范围、并发数、总超时时间和结果数量；默认只允许 loopback，避免成为内网或公网扫描工具。
- 容量类配置必须有默认值、单项硬上限和总内存预算，避免多实例或多订阅者组合造成不可控内存占用。
- 错误信息不得泄露无关环境变量、凭据或系统敏感路径。

## 推荐调用流程

### 串口一问一答

```text
instance_create(type=Serial)
-> serial_config
-> port_connect
-> port_send
-> port_pull
-> port_disconnect
-> instance_release
```

### TCP/UDP 基础调试

```text
instance_create(type=TCP|UDP)
-> tcp_udp_config
-> port_connect
-> port_send
-> port_pull
-> instance_query
-> port_disconnect
```

### 多 AI 流式观察

```text
instance_create
-> serial_config/tcp_udp_config
-> port_connect
-> port_subscribe_stream
-> port_send
-> notifications/resources/updated
-> port_unsubscribe_stream
-> port_disconnect
```

## 初版验收标准

- 可以创建串口、TCP、UDP 三类实例，并通过 `instance_list` 与 `instance_query` 查询到稳定状态。
- 多个实例的配置、缓冲区、收发计数和错误状态互不污染。
- 串口资源独占逻辑可验证：同一串口不能被两个已连接实例同时占用。
- TCP/UDP 基础收发可通过本地 loopback 或测试服务验证。
- `port_send` 并发调用时，同一实例的数据帧按队列顺序写入，不交织。
- `port_pull` 读取后普通 `rx` 缓冲区减少或清空，体现消耗式读取语义。
- 多个 AI 会话订阅同一实例时，可收到同一份接收数据通知，且不影响 `port_pull` 缓冲区语义。
- `port_disconnect` 后底层资源释放，后台任务停止，实例状态变为 `Disconnected`。
- `instance_release` 后再次操作该句柄返回 `HandleNotFound`。
- 主要错误类别均有可预测、结构化、可读的返回。

## 与进阶能力的边界

初版实现应保留扩展点，但不要把进阶能力做成隐式半成品：

- 协议辅助可以通过 `data` 的 `text`/`hex` 编码满足基础调试，不在初版实现 Modbus helper。
- 最小订阅只保证 `on_receive` 广播，不承诺正则、Hex 组帧或按行触发。
- 状态模型保留后台任务字段，但初版后台任务只包含收发和订阅广播。
- 错误模型保留扩展类别，但初版只要求覆盖基础连接和收发链路。
