# 05 端口 MCP 状态机与资源生命周期

本文档承接 [01-初版工程功能设计大纲](01-初版工程功能设计大纲.md)、[03-初版工具契约](03-初版工具契约.md) 与 [04-运行时架构设计](04-运行时架构设计.md)，固化初版实例状态机、工具允许状态、状态迁移动作、资源锁生命周期、断开释放清理顺序和错误态恢复规则。

本文档只定义状态与资源生命周期。发送队列容量、接收缓冲区溢出策略、订阅广播通道细节由 `06-并发与缓冲区设计.md` 承接；完整错误码、日志字段和用户消息由 `07-错误模型与返回格式.md` 承接。

## 设计结论

- 初版状态集合固定为 `Created`、`Configured`、`Connected`、`Disconnecting`、`Disconnected`、`Released`、`Error`。
- 连接打开失败、后台接收任务致命失败、后台发送任务致命失败、断开清理失败统一进入 `Error`。
- `Error` 是可查询、可断开、可释放的故障收敛状态，不允许继续发送、拉取或订阅新流。
- `port_disconnect` 对 `Disconnected` 实例幂等成功，状态保持 `Disconnected`。
- `instance_release(force=true)` 对 `Connected` 或 `Error` 实例立即释放句柄，后台继续执行底层资源关闭。
- 强制释放时句柄同步失效，但底层资源锁不得直接变为可复用；锁必须进入 `closing` tombstone 状态，直到后台 close 成功或达到明确超时后才释放。
- `Released` 不是实例表中的可查询状态；工具返回 `Released` 后，同一 `handle_id` 后续操作必须返回 `HandleNotFound`。

## 状态定义

| 状态 | 是否在实例表中 | 是否占用底层资源 | 是否可配置 | 是否可收发 | 说明 |
| --- | --- | --- | --- | --- | --- |
| `Created` | 是 | 否 | 是 | 否 | 已创建句柄，但配置不完整。 |
| `Configured` | 是 | 否 | 是 | 否 | 配置完整，可以尝试连接。 |
| `Connected` | 是 | 是 | 否 | 是 | 底层资源已打开，接收/发送任务已启动。 |
| `Disconnecting` | 是 | 可能 | 否 | 否 | 正在取消任务、关闭 transport、释放资源锁。 |
| `Disconnected` | 是 | 否 | 是 | 否 | 连接已关闭，配置和统计保留，可再次连接。 |
| `Error` | 是 | 可能 | 否 | 否 | 连接、读写或清理发生致命错误，需要查询、断开或释放。 |
| `Released` | 否 | 否或后台清理中 | 否 | 否 | 句柄已失效，不再出现在实例表中；相关资源锁可能短暂处于 `closing` tombstone，阻止同一资源立即复用。 |

`Released` 用于成功返回和文档表达，不应作为实例表中长期保存的状态。

## 总体状态图

```text
instance_create
  -> Created

Created
  -> Configured      by serial_config / tcp_udp_config
  -> Released        by instance_release

Configured
  -> Configured      by serial_config / tcp_udp_config overwrite
  -> Connected       by port_connect success
  -> Error           by port_connect failure
  -> Released        by instance_release

Connected
  -> Disconnecting   by port_disconnect
  -> Error           by receive/send task fatal error
  -> Released        by instance_release(force=true)

Disconnecting
  -> Disconnected    by cleanup success
  -> Error           by cleanup timeout/failure
  -> Released        by force release during cleanup

Disconnected
  -> Configured      by serial_config / tcp_udp_config overwrite
  -> Connected       by port_connect success
  -> Error           by port_connect failure
  -> Released        by instance_release

Error
  -> Disconnecting   by port_disconnect
  -> Released        by instance_release
  -> Released        by instance_release(force=true)
```

## 工具允许状态矩阵

本文档以 [03-初版工具契约](03-初版工具契约.md) 为准，并补齐边界选择。

| 工具 | 允许状态 | 成功后状态 | 说明 |
| --- | --- | --- | --- |
| `instance_create` | 无前置状态 | `Created` | 创建实例表记录，不打开底层资源。 |
| `instance_list` | 无前置状态 | 不改变 | 只列出未释放实例。 |
| `instance_query` | `Created`、`Configured`、`Connected`、`Disconnecting`、`Disconnected`、`Error` | 不改变 | `Released` 返回 `HandleNotFound`。 |
| `instance_use` | `Created`、`Configured`、`Connected`、`Disconnected`、`Error` | 不改变 | 不允许绑定 `Disconnecting` 或 `Released`。 |
| `instance_release` | `Created`、`Configured`、`Disconnected`、`Error` | `Released` | 清理绑定、订阅、缓冲区和队列后移出实例表。 |
| `instance_release(force=true)` | `Created`、`Configured`、`Connected`、`Disconnecting`、`Disconnected`、`Error` | `Released` | 句柄立即失效；仍在关闭中的底层资源锁进入 `closing` tombstone，底层 close 后台执行。 |
| `serial_config` | `Created`、`Configured`、`Disconnected` | `Configured` | 不检查串口占用，不打开串口。 |
| `tcp_udp_config` | `Created`、`Configured`、`Disconnected` | `Configured` | 不检查监听冲突，不创建 socket。 |
| `port_scan` | 无前置状态 | 不改变 | 不依赖实例状态。 |
| `port_connect` | `Configured`、`Disconnected` | `Connected` 或 `Error` | 申请资源锁、打开 transport、启动任务。 |
| `port_disconnect` | `Connected`、`Error`、`Disconnected` | `Disconnected` 或 `Error` | `Disconnected` 幂等成功。 |
| `port_send` | `Connected` | `Connected` 或 `Error` | 入队或写入失败可能使实例进入 `Error`。 |
| `port_pull` | `Connected` | `Connected` | 超时不改变状态。 |
| `port_clear` | `Configured`、`Connected`、`Disconnected`、`Error` | 不改变 | `Created` 返回 `InvalidState`，避免空清理语义分歧。 |
| `port_subscribe_stream` | `Connected` | `Connected` | 同一会话重复订阅幂等成功。 |
| `port_unsubscribe_stream` | `Connected`、`Disconnected`、`Error` | 不改变 | 未订阅时幂等成功。 |

## 状态迁移动作

### 创建实例

```text
instance_create
-> validate type
-> allocate handle_id
-> create RuntimeInstance(state=Created)
-> insert registry.instances
```

创建阶段不得占用串口、监听端口、UDP bind 地址或启动后台任务。

### 写入配置

```text
serial_config / tcp_udp_config
-> check instance exists
-> check current state in Created | Configured | Disconnected
-> validate args
-> write config snapshot
-> clear config-related last_error if any
-> state = Configured
```

配置覆盖必须是原子的：参数校验失败时保留旧配置和旧状态。

### 连接成功

```text
port_connect
-> check state in Configured | Disconnected
-> normalize resource key
-> acquire resource lock
-> open concrete transport
-> create tx queue / rx buffer handles / task group
-> start receive task and send task
-> set connected_at and last_activity_at
-> state = Connected
```

连接成功的提交点是 `state = Connected`。在此之前如任何步骤失败，必须执行连接失败回滚。

### 连接失败回滚

```text
port_connect failure
-> cancel any started task
-> close partially opened transport if present
-> release acquired resource lock
-> record last_error
-> state = Error
```

即使底层资源已经清理成功，初版仍统一进入 `Error`，让 AI 可以通过 `instance_query` 看到失败原因，并显式选择重新配置、断开或释放。

### 后台任务致命失败

接收任务或发送任务遇到不可恢复错误时：

```text
task fatal error
-> report task exit reason
-> stop sibling task
-> close tx queue entrance
-> record last_error
-> state = Error
```

致命错误包括底层连接中断、不可恢复读写失败、任务 panic 被捕获、发送队列永久关闭等。普通 `ReadTimeout` 不属于致命错误，不改变状态。

### 断开连接

```text
port_disconnect on Connected | Error
-> state = Disconnecting
-> cancel receive/send tasks
-> close tx queue entrance
-> wait task group exit until timeout
-> close transport
-> release resource lock
-> invalidate subscriptions
-> state = Disconnected
```

断开成功后保留配置、创建时间、最近连接时间、统计、最后错误摘要和普通 `rx` 缓冲区。发送队列中未写入项必须被丢弃并计入清理结果或日志。

### 断开失败

```text
disconnect cleanup failure
-> best-effort release resource lock if safe
-> record last_error
-> state = Error
```

断开失败后实例仍在实例表中。用户可再次调用 `port_disconnect` 或 `instance_release(force=true)`。

### 幂等断开

```text
port_disconnect on Disconnected
-> return ok=true
-> state remains Disconnected
```

幂等成功不应清空配置、统计或缓冲区，也不应改变会话绑定。

### 普通释放

```text
instance_release on Created | Configured | Disconnected | Error
-> reject if Connected and force=false
-> cancel subscription records
-> clear session bindings pointing to handle_id
-> clear tx queue and rx buffer
-> release resource lock if still held
-> remove from registry.instances
-> return state=Released
```

对 `Error` 状态执行普通释放时，允许 best-effort 清理仍残留的任务或 transport。释放返回成功后，同一句柄不可再查询。

### 强制释放

```text
instance_release(force=true)
-> mark handle as releasing
-> remove from registry.instances
-> clear session bindings and subscriptions
-> move held resource lock to closing tombstone with generation and deadline
-> spawn background close task for task group and transport
-> release tombstone after close success, or mark stale_close when configured close deadline expires
-> return state=Released immediately
```

强制释放的用户语义是句柄立即失效。后台 close 结果只能写入日志或调试记录，不能再通过原 `handle_id` 返回给用户。

## 资源生命周期

### 资源分类

| 资源 | 创建时机 | 释放时机 | 所属状态 |
| --- | --- | --- | --- |
| `handle_id` | `instance_create` | `instance_release` | `Created` 到 `Error` |
| 配置快照 | `serial_config` / `tcp_udp_config` | `instance_release` | `Configured` 到 `Error` |
| 串口独占锁 | `port_connect` 打开串口前 | disconnect/release 清理阶段 | `Connected`、`Disconnecting`、可能 `Error` |
| TCP listen 锁 | `port_connect` bind/listen 前 | disconnect/release 清理阶段 | `Connected`、`Disconnecting`、可能 `Error` |
| UDP bind 锁 | `port_connect` bind 前 | disconnect/release 清理阶段 | `Connected`、`Disconnecting`、可能 `Error` |
| Transport 句柄 | 资源锁申请成功后 | disconnect/release close 阶段 | `Connected`、`Disconnecting`、可能 `Error` |
| 接收/发送任务 | transport 打开成功后 | disconnect/release 取消阶段 | `Connected`、`Disconnecting`、可能 `Error` |
| 发送队列 | connect 成功前准备 | disconnect/release 清理阶段 | `Connected`、`Disconnecting`、可能 `Error` |
| 普通 `rx` 缓冲区 | 实例创建时或连接前初始化 | release 清理阶段 | 未释放期间 |
| 订阅记录 | `port_subscribe_stream` | unsubscribe/disconnect/release | `Connected`、断开时失效 |
| 会话绑定 | `instance_use` | release 清理绑定 | 未释放期间 |

### 资源锁规则

资源锁是防止重复占用底层资源的运行时约束，不是底层资源本身。

| 场景 | 锁处理 |
| --- | --- |
| 连接前参数非法 | 不申请锁。 |
| 资源锁已被占用 | 返回 `ResourceBusy`，状态进入 `Error`。 |
| 锁申请成功但 transport 打开失败 | 释放锁，状态进入 `Error`。 |
| `port_disconnect` 成功 | 释放锁后进入 `Disconnected`。 |
| `port_disconnect` 清理失败 | 尽力释放锁，状态进入 `Error`。 |
| `instance_release(force=true)` | 句柄立即失效；锁进入 `closing` tombstone 并阻止同一资源复用。close 成功后释放 tombstone；deadline 到达但 close 未确认时标记 `stale_close=true` 后释放 tombstone。 |

资源锁 key 由 [04-运行时架构设计](04-运行时架构设计.md) 定义：串口使用规范化串口名，TCP listen 使用监听地址端口，UDP bind/listen 使用绑定地址端口。

### 底层资源关闭

底层 transport close 分为同步路径和后台路径：

- `port_disconnect`：优先同步等待任务退出和 transport close，成功后返回 `Disconnected`。
- `instance_release(force=true)`：句柄立即释放，transport close 转入后台 best-effort；资源锁以 `closing` tombstone 形式保留隔离，直到后台 close 成功或达到配置的 close deadline。
- 后台 close 失败不得恢复句柄，也不得把旧资源静默交给新实例；只能记录日志或调试事件，并在 tombstone 超时释放时留下 `background_close_error` 诊断事件和 `stale_close=true` 标记。
- tombstone 因 deadline 释放后，新实例仍必须重新执行真实底层 open/bind。若 OS 或驱动仍报告资源占用，应返回稳定 `ResourceBusy/RESOURCE_LOCK_STALE` 或对应具体 busy code，并可重新建立短 tombstone/backoff，避免快速重试打满日志或驱动。

此策略优先保证 AI 调用侧不会卡死在坏连接上，同时避免旧 transport 尚未关闭时同一物理资源被新实例复用。

## 订阅生命周期

订阅依附于 `handle_id + session_id`，不是独立实例。

| 事件 | 订阅处理 |
| --- | --- |
| `port_subscribe_stream` 成功 | 添加或更新当前会话订阅记录。 |
| 同一会话重复订阅 | 幂等成功，返回同一 stream URI 或更新限制参数。 |
| `port_unsubscribe_stream` | 删除当前会话订阅；未订阅时幂等成功。 |
| `port_disconnect` 成功 | 当前实例所有订阅停止并失效。 |
| 后台任务进入 `Error` | 当前实例所有订阅停止并标记错误结束。 |
| `instance_release` | 删除当前实例所有订阅记录。 |

断开后是否需要用户显式取消订阅：不需要。`port_unsubscribe_stream` 仍允许在 `Disconnected` 或 `Error` 中调用，作为幂等清理入口。

## 会话绑定生命周期

会话绑定只记录默认实例，不代表资源所有权。

| 事件 | 会话绑定处理 |
| --- | --- |
| `instance_use` | 当前会话绑定到目标 `handle_id`。 |
| `port_disconnect` | 绑定保留，便于重连后继续使用。 |
| 实例进入 `Error` | 绑定保留，便于查询错误。 |
| `instance_release` | 删除所有指向该 `handle_id` 的绑定。 |
| 后续使用已释放绑定 | 应返回 `HandleNotFound` 或要求重新 `instance_use`，具体错误包装由 `07` 定义。 |

`instance_use` 不允许绑定 `Disconnecting` 实例，避免用户把默认目标指向正在清理的临时状态。

## 原子性与锁顺序

为避免死锁和状态撕裂，初版实现应遵循固定顺序：

1. 获取全局实例表引用。
2. 获取目标实例状态锁，检查前置状态。
3. 如需长时间 I/O，释放实例状态锁。
4. 申请或释放资源锁。
5. 执行 transport open/close 或任务等待。
6. 重新获取实例状态锁，提交状态和统计变化。

禁止在持有实例状态锁时执行长时间串口读写、TCP 连接、UDP bind、任务 join 或后台 close。

状态提交应满足：

- 工具失败时不得留下半写配置。
- 连接失败时不得留下已占用资源锁。
- 释放成功时不得留下可查询实例。
- 后台任务进入 `Error` 时必须写入 `last_error`。

## 错误态恢复规则

`Error` 状态只允许以下恢复入口：

| 工具 | 结果 |
| --- | --- |
| `instance_query` | 查看配置、统计、缓冲区长度、订阅数量和 `last_error`。 |
| `instance_use` | 允许绑定，便于当前会话继续查询或释放。 |
| `port_disconnect` | 尝试取消任务、关闭 transport、释放资源锁，成功后进入 `Disconnected`。 |
| `port_clear` | 清理普通缓冲区或未写入发送项，状态仍为 `Error`。 |
| `port_unsubscribe_stream` | 清理当前会话订阅，状态仍为 `Error`。 |
| `instance_release` | 移出实例表，句柄失效。 |

`Error` 状态不允许直接重新配置或重新连接。推荐流程：

```text
instance_query
-> port_disconnect
-> serial_config / tcp_udp_config if needed
-> port_connect
```

如果 `port_disconnect` 失败或用户不关心保留实例，可执行：

```text
instance_release(force=true)
-> instance_create
-> config
-> connect
```

## 边界选择

### `port_clear` 在 `Created` 状态

`Created` 没有有效配置、发送队列或接收缓冲区语义。初版选择返回 `InvalidState`，而不是空清理成功，避免测试和用户预期分裂。

### `port_disconnect` 在 `Disconnected` 状态

初版选择幂等成功，原因是断开常用于清理流程，重复调用不应制造额外失败。

### 连接失败进入 `Error`

即使连接失败时可以回到 `Configured` 或 `Disconnected`，初版仍统一进入 `Error`。这样能保留失败现场，避免 AI 在未查看错误原因时无意重复重连。

### 强制释放立即失效句柄

强制释放优先满足“不要让坏连接阻塞用户继续工作”。句柄立即失效，底层 close 后台执行；但资源锁进入 `closing` tombstone，不允许同一物理资源在旧 close 完成前被新实例复用。后台 close 失败后不得重新注册旧句柄，只能留下诊断事件并按配置 deadline 处理 tombstone。

## 设计验收标准

本文档完成后，初版工程应能据此实现和测试：

- 每个初版工具在每个实例状态下是允许、拒绝还是幂等成功。
- 连接成功、连接失败、后台任务失败、断开成功、断开失败、释放成功的状态迁移。
- 串口、TCP listen、UDP bind 资源锁的申请、释放和失败回滚。
- `Error` 状态下可查询、可断开、可清理、可释放，但不可继续收发。
- `instance_release(force=true)` 句柄立即失效，资源锁进入 `closing` tombstone，底层 close 后台完成或超时后才允许同资源复用。
- 断开、释放、订阅、会话绑定之间的清理关系。
