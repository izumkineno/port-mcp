## 阅读顺序

1. 先读 [01-初版工程功能设计大纲](01-初版工程功能设计大纲.md)，确认初版必须实现什么、不实现什么。
2. 再读 [02-进阶实现大纲](02-进阶实现大纲.md)，了解扩展能力如何接续初版架构。
3. 再读 [03-初版工具契约](03-初版工具契约.md)，确认初版工具的调用边界。
4. 再读 [04-运行时架构设计](04-运行时架构设计.md)，确认 Rust 原生 MCP server、运行时分层和后台任务模型。
5. 再读 [05-状态机与资源生命周期](05-状态机与资源生命周期.md)，确认状态转移、资源锁和断开释放清理顺序。

7. 再读 [07-错误模型与返回格式](07-错误模型与返回格式.md)，确认统一返回、错误码、日志、脱敏和恢复建议。
8. 再读 [08-测试与验收计划](08-测试与验收计划.md)，确认自动化测试、loopback 测试、串口手工验收和平台门槛。
9. 再读 [09-库选择与依赖决策](09-库选择与依赖决策.md)，确认编码前的库选择、依赖边界和替代方案。
