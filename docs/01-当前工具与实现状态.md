# 01 当前工具与实现状态

本文档是 `docs/1.0.0文档归档/` 之后的当前主文档入口，用于汇总当前 Rust `port-mcp` 的 MCP 工具面、实现状态和验证状态。1.0.0 归档仍作为设计、施工和历史验收证据；本文只记录当前主要能力，不重复展开全部历史设计。

## 当前结论

当前 `port-mcp` 是 Rust 原生 stdio MCP server，面向 Serial、TCP、UDP 和可选 VISA 仪器/端口调试场景。代码中权威工具入口位于 `src/mcp/tools.rs` 的 `#[tool_router] impl PortMcpServer`，当前公开 21 个 MCP 工具。

这些工具已经覆盖：实例生命周期、连接配置、资源扫描、连接/断开、发送/拉取、清缓冲、订阅状态、原始 I/O 日志预览，以及轻量协议辅助。

## 工具分组

| 分组 | 工具 | 当前用途 |
| --- | --- | --- |
| 使用指引 | `usage_guide` | 返回机器可读 quickstart、常见调用序列和工具注意事项。 |
| 实例管理 | `instance_create`, `instance_list`, `instance_query`, `instance_use`, `instance_release` | 创建、列出、查询、绑定默认实例和释放实例。 |
| 连接配置 | `serial_config`, `tcp_udp_config`, `visa_config` | 配置 Serial、TCP client/listen、UDP、VISA resource address 与超时/编码参数。 |
| 资源扫描 | `port_scan` | 扫描 Serial 端口、VISA resource，以及 loopback TCP/UDP 端口范围。 |
| 端口行为 | `port_connect`, `port_disconnect`, `port_send`, `port_pull`, `port_clear` | 打开/关闭资源，发送 text/hex payload，拉取接收摘要，清理 tx/rx/all 缓冲。 |
| 流式订阅 | `port_subscribe_stream`, `port_unsubscribe_stream` | 记录当前 MCP session 对连接实例的接收流订阅关系。 |
| 调试日志 | `debug_log_config` | 控制 `port_send` / `port_pull` 原始 I/O 预览日志字节数，`0` 表示关闭。 |
| 编码辅助 | `str_to_hex`, `hex_to_str` | 在文本和十六进制表示之间转换，并执行输入限制与校验。 |
| 协议辅助 | `modbus_helper`, `scpi_helper`, `at_helper`, `slip_helper` | 提供 Modbus RTU 打包/解包、SCPI 归一化、AT 分类、SLIP 编解码。 |

## 实现状态

| 能力 | 当前状态 | 说明 |
| --- | --- | --- |
| MCP stdio server | 已实现 | 使用 `rmcp` 宏注册工具并返回结构化 `CallToolResult`。 |
| 实例生命周期 | 已实现 | `RuntimeRegistry` 负责状态机、资源锁、会话默认实例和释放语义。 |
| Serial | 已实现 | 支持串口扫描、参数配置、真实 `serialport` worker、读写和关闭。 |
| TCP client | 已实现 | 支持 loopback/允许范围内连接、发送、读取和关闭。 |
| TCP listen | 已实现 | 支持监听、multi-client peer 摘要、指定 peer 发送和广播发送。 |
| UDP | 已实现 | 支持 bind、remote endpoint、datagram 收发和关闭。 |
| VISA | 可选实现 | `visa` Cargo feature 启用时接入 `visa-rs`；未启用时返回 feature-not-compiled 错误。 |
| 缓冲与计数 | 已实现 | 支持 tx/rx 计数、pull 摘要、容量限制和 clear。 |
| 流式订阅状态 | 已实现 | 运行时记录订阅关系并隔离慢订阅者；客户端可见通知行为需按 MCP 客户端单独验证。 |
| 原始 I/O 日志预览 | 已实现 | 默认不输出原始 payload，可通过 `debug_log_config` 显式开启有限预览。 |
| 协议 helper | 已实现 | 当前已包含 text/hex、Modbus RTU、SCPI、AT、SLIP 轻量辅助。 |

## 验证状态

最近一次默认验证：

```powershell
rtk cargo test
```

结果：默认 feature 下 98 个 Rust 测试通过。

当前自动化覆盖主要包括：

- runtime 生命周期、状态机、资源锁、缓冲区、队列、订阅与错误路径。
- MCP 工具 smoke 与参数/返回外形。
- TCP listen 多客户端、TCP client loopback echo、UDP datagram roundtrip。
- encoding、Modbus RTU、SCPI、AT、SLIP helper 单元测试。
- VISA 类型、配置与 feature-gated 错误路径；真实 VISA driver/runtime 需要在启用 `visa` feature 且本机具备 VISA 环境时额外验证。
