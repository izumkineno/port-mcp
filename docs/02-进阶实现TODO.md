## P1 高级流式订阅

目标：把当前最小订阅状态扩展为可用于文本日志、高频数据和二进制协议的触发体系，同时保持通知限速和订阅者隔离。

- [ ] `line` 触发：按 `\n` / `\r\n` 聚合完整行后通知。
- [ ] `batch` 触发：按时间窗口或字节窗口聚合通知，优先解决高频通知风暴。
- [ ] `match_hex` 触发：匹配固定字节序列，适配二进制帧头或特征字节。
- [ ] `match_regex` 触发：支持受限正则，必须有输入长度、复杂度和执行超时保护。
- [ ] `frame_hex` 组帧：按帧头、长度字段和可选校验策略组装二进制帧。
- [ ] `instance_query` 或订阅查询路径可观察订阅条件、命中次数和丢弃/合并次数。

验收门槛：

- [ ] 同一实例上的多个订阅者可使用不同触发条件。
- [ ] 高频输入不会造成无界通知或内存增长。
- [ ] 正则和组帧路径有明确错误码、超时和容量上限。
- [ ] 二进制通知可保留 hex 表达，不被 UTF-8 文本解码破坏。

## P1 数据安全与性能治理

目标：在进阶能力扩张前先补齐横向保护，防止订阅、探测、导出或 helper 把系统推向不可控。

- [ ] 统一 helper、组帧、订阅匹配和探测输入大小上限。
- [ ] 增加 per-session 或 per-instance 订阅通知速率治理。
- [ ] 增加长连接日志采样或滚动策略，避免调试日志无限增长。
- [ ] 网络扫描或受控探测继续默认限制在 loopback；任何 allowlist 扩展必须显式配置、可审计、可 dry-run。
- [ ] 为路径、凭据形态字符串和高风险 payload 增加脱敏策略。
- [ ] 为大型返回结果建立统一 `ResultTooLarge` 或等价错误边界。

验收门槛：

- [ ] 恶意或超大输入不能导致 CPU、内存或日志无限放大。
- [ ] 默认配置不扩大网络暴露面。
- [ ] 安全边界和错误返回可通过自动化测试覆盖。

## P2 调试记录导出

目标：把一次端口调试过程导出成可复现、可审计、默认脱敏的材料包。

- [ ] 定义导出内容：会话元数据、实例摘要、工具调用、收发摘要、错误链和验证命令。
- [ ] 定义附件结构：原始 payload、截图、驱动信息或外部日志只能作为受控附件。
- [ ] 增加脱敏报告：说明哪些字段被脱敏、哪些原始材料被排除。
- [ ] 生成复现实验脚本或最小调用序列。
- [ ] 对导出大小、附件数量和敏感字段建立限制。

验收门槛：

- [ ] 无真实硬件场景可导出 mock/TCP/UDP loopback 调试记录。
- [ ] 导出包默认不包含未脱敏原始敏感材料。
- [ ] 导出材料足以复现一次失败或成功路径。

## P2 链路维持与自动重连

目标：支持长连接调试中的心跳、断线诊断和受控重连，但不绕过现有发送队列和生命周期。

- [ ] `keep_alive_config`：启用、更新、停止心跳任务。
- [ ] 心跳 payload 必须进入现有发送路径，避免与用户发送帧无序交织。
- [ ] 心跳统计与用户发送统计分离。
- [ ] `reconnect_config`：配置启用状态、最大次数、退避、抖动和停止条件。
- [ ] `instance_query` 可见重连次数、最后成功时间、最后失败错误和停止原因。
- [ ] 主动 `port_disconnect`、`instance_release` 或 force release 时停止后台任务。

验收门槛：

- [ ] 心跳可启停且不会绕过 payload 限制。
- [ ] 自动重连达到上限后进入可诊断状态，不无限重试。
- [ ] 用户主动断开后不会继续后台重连。

## P2 UDP Datagram 元数据增强

目标：在保持普通 `port_pull` 字节流兼容的前提下补足 UDP 来源和 datagram 边界可见性。

- [ ] 在接收路径保留 datagram 来源地址、接收时间、payload 长度和截断状态。
- [ ] 增加最近来源查询或在 `instance_query` 中展示来源摘要。
- [ ] 支持按来源过滤拉取或订阅。
- [ ] 增加多来源统计：来源数量、每来源接收字节、最后接收时间。
- [ ] 调试日志和导出材料能标记 UDP source metadata。

验收门槛：

- [ ] 普通 `port_pull` 兼容现有返回摘要。
- [ ] 多来源 UDP 场景下可区分来源并定位回包。
- [ ] 来源元数据不引入无界内存增长。

## P3 VISA 后续增强

目标：在现有 feature-gated VISA 基础上补齐真实环境验收和仪器识别体验。

- [ ] 增加可选 `*IDN?` 识别流程，失败不影响保存配置或基础连接。
- [ ] 明确 `query_idn_on_connect` 的最终行为和返回位置。
- [ ] 增加真实 VISA runtime / driver 环境下的手工验收记录模板。
- [ ] 细化 VISA runtime 缺失、resource 不存在、资源占用、读写超时的恢复建议。
- [ ] 评估 `visa` feature 在 CI 中的可验证边界，避免默认构建依赖真实驱动。

验收门槛：

- [ ] 默认 feature 不受 VISA 环境影响。
- [ ] 启用 `visa` feature 且有真实环境时可完成一次 SCPI 一问一答。
- [ ] 缺驱动或缺设备时错误信息可诊断。

## P3 协议 Helper 后续增强

目标：在当前轻量 helper 基础上补足明确有价值、可纯计算或受控探测的增强。

- [ ] Modbus ASCII/LRC pack/unpack。
- [ ] SCPI 响应摘要增强，但不替代仪器手册和厂商专用解析。
- [ ] AT 响应分类增强，避免硬编码厂商私有扩展为通用契约。
- [ ] SLIP 与后续组帧能力对齐，明确非法帧、半包和转义错误边界。
- [ ] 受控 `device_port_probe` 类辅助：只允许有限候选集合、受控并发、总超时和审计记录。
- [ ] `response_matcher` 最小匹配规则：固定字节、文本片段或 checksum 通过。

验收门槛：

- [ ] helper 输出仍然通过 `port_send` / `port_pull` 主链路使用。
- [ ] 所有 helper 有输入大小上限和结构化错误。
- [ ] 探测类能力不是通用网络扫描器，默认不扩大网络边界。

## P3 异常注入

目标：为容错测试和现场问题复现提供可控的故障模拟能力。

- [ ] `inject_error` 或等价配置工具：启用、查询、停止和清理注入状态。
- [ ] `disconnect`：模拟目标实例断连。
- [ ] `delay`：模拟读写延迟。
- [ ] `drop_rx`：丢弃部分接收数据。
- [ ] `corrupt_tx`：发送前改写部分字节。
- [ ] `bad_checksum`：配合协议 helper 生成错误校验帧。
- [ ] 调试记录和导出材料标记受注入影响的数据或错误。

验收门槛：

- [ ] 注入只影响目标实例，不污染其他实例。
- [ ] 注入行为可观察、可停止、可清理。
- [ ] 注入错误不会伪装成普通设备错误。
