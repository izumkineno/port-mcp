# 03 端口 MCP 初版工具契约

本文档承接 [01-初版工程功能设计大纲](01-初版工程功能设计大纲.md)，把初版工具固定为可实现、可测试、可审查的功能级契约。本文档定义每个工具的用途、入参、成功返回、状态要求、错误类别和边界条件。

本文档不定义底层模块目录、锁实现、队列数据结构、完整错误码、测试脚本或运行配置；这些内容分别由后续 `04` 到 `08` 文档承接。

## 契约范围

初版工具分为四组：

| 分组 | 工具 |
| --- | --- |
| 实例管理 | `instance_create`、`instance_list`、`instance_query`、`instance_use`、`instance_release` |
| 连接配置 | `serial_config`、`tcp_udp_config` |
| 端口行为 | `port_scan`、`port_connect`、`port_disconnect`、`port_send`、`port_pull`、`port_clear` |
| 最小流式订阅 | `port_subscribe_stream`、`port_unsubscribe_stream` |

## 通用约定

### 命名与标识

- `handle_id` 是连接实例的唯一标识，由 `instance_create` 生成。
- 初版建议使用可读前缀：串口 `h_ser_001`，TCP `h_tcp_001`，UDP `h_udp_001`。
- 工具名使用小写蛇形命名。
- 工具入参使用小写蛇形字段名。

### 连接类型

| 类型 | 含义 |
| --- | --- |
| `Serial` | 本机串口通信实例。 |
| `TCP` | 面向连接的 TCP socket 实例。 |
| `UDP` | UDP datagram 通信实例。 |

### 状态名

初版工具只引用以下实例状态：

| 状态 | 含义 |
| --- | --- |
| `Created` | 实例已创建，尚未配置。 |
| `Configured` | 实例配置完整，可以连接。 |
| `Connected` | 底层资源已打开，可以收发。 |
| `Disconnecting` | 正在断开并清理资源。 |
| `Disconnected` | 连接已关闭，配置保留。 |
| `Released` | 实例已释放，不应再被查询或操作。 |
| `Error` | 实例发生连接、读写或后台任务错误。 |

完整状态转移和清理顺序由 `05-状态机与资源生命周期.md` 定义。本文档只规定各工具允许在哪些状态调用。

### 时间、数量与编码

- 所有超时字段统一使用毫秒，字段名使用 `timeout_ms`。
- 所有字节数量字段使用非负整数，字段名使用 `max_bytes`、`sent_bytes`、`received_bytes` 等。
- 数据编码字段使用 `encoding`，初版支持 `text` 和 `hex`。
- 文本编码的字符集默认 `UTF-8`；如需配置字符集，使用 `text_encoding` 字段。

### 成功返回

成功返回至少包含：

| 字段 | 含义 |
| --- | --- |
| `ok` | 固定为 `true`。 |
| `tool` | 当前工具名。 |
| `handle_id` | 与实例相关的工具必须返回解析后的实例句柄；全局工具可省略。 |
| `state` | 操作后的实例状态；不改变状态的工具可返回当前状态。 |
| `data` | 工具特定结果。 |

示例：

```json
{
  "ok": true,
  "tool": "port_send",
  "handle_id": "h_tcp_001",
  "state": "Connected",
  "data": {
    "sent_bytes": 6,
    "queued": true
  }
}
```

### 失败返回

失败返回至少包含：

| 字段 | 含义 |
| --- | --- |
| `ok` | 固定为 `false`。 |
| `tool` | 当前工具名。 |
| `handle_id` | 如果请求中包含句柄，则原样返回。 |
| `error.category` | 错误类别。 |
| `error.message` | 面向 AI 和用户的可读错误。 |
| `error.recovery_hint` | 可恢复建议。 |

示例：

```json
{
  "ok": false,
  "tool": "port_connect",
  "handle_id": "h_ser_002",
  "error": {
    "category": "ResourceBusy",
    "message": "Serial port COM3 is already owned by h_ser_001.",
    "recovery_hint": "Disconnect the owning instance or choose another port."
  }
}
```

详细错误码、日志格式和内部错误映射由 `07-错误模型与返回格式.md` 定义。

### 实例句柄解析

与实例相关的工具优先使用请求中显式传入的 `handle_id`。若某工具将 `handle_id` 标记为可选且请求未传入，则服务端必须按当前 `session_id` 解析会话默认实例，并在成功返回中填充实际使用的 `handle_id`。

未传 `handle_id` 且当前会话没有默认实例时，返回 `HandleNotFound/SESSION_BINDING_MISSING`。MCP SDK 或运行模式无法提供稳定 `session_id` 时，所有依赖会话身份的缺省解析路径、`instance_use`、`port_subscribe_stream` 和 `port_unsubscribe_stream` 必须返回 `InvalidState/SESSION_ID_UNAVAILABLE`；显式传入 `handle_id` 且不依赖会话身份的工具仍可使用。

### 通用错误类别

| 错误类别 | 含义 |
| --- | --- |
| `HandleNotFound` | `handle_id` 不存在或已释放。 |
| `InvalidState` | 当前状态不允许执行该工具。 |
| `InvalidArgument` | 参数缺失、类型错误、枚举非法或范围越界。 |
| `ResourceBusy` | 串口、监听地址或其他资源被占用。 |
| `ConnectTimeout` | 连接或打开底层资源超时。 |
| `ReadTimeout` | 读取在超时时间内没有得到数据。 |
| `WriteFailed` | 写入失败、连接中断或发送队列拒绝。 |
| `BufferLimitExceeded` | 缓冲区、返回数据或通知 payload 超过限制。 |

## 实例管理工具

### `instance_create`

用途：创建一个新的连接实例，并返回全局唯一 `handle_id`。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `type` | 是 | `Serial`、`TCP` 或 `UDP`。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `handle_id` | 新实例句柄。 |
| `state` | `Created`。 |
| `data.type` | 实例类型。 |
| `data.created_at` | 创建时间戳。 |

状态要求：无前置状态。

错误类别：`InvalidArgument`。

边界条件：

- 不允许创建 `Visa` 或进阶类型实例。
- `handle_id` 不应复用仍可查询的旧句柄。
- 创建实例不应打开串口或网络 socket。

### `instance_list`

用途：列出当前所有未释放实例的摘要。

入参：无。

成功返回：

| 字段 | 说明 |
| --- | --- |
| `data.instances` | 实例摘要数组。 |

实例摘要至少包含：

| 字段 | 说明 |
| --- | --- |
| `handle_id` | 实例句柄。 |
| `type` | 连接类型。 |
| `state` | 当前状态。 |
| `resource` | 串口名、远端地址或监听地址摘要。 |
| `created_at` | 创建时间戳。 |
| `connected_at` | 最近连接时间，可为空。 |
| `tx_bytes` | 已发送字节数。 |
| `rx_bytes` | 已接收字节数。 |

状态要求：无前置状态。

错误类别：无业务错误；内部异常按实现统一包装。

边界条件：

- `Released` 实例不出现在列表中。
- 不应返回完整敏感环境信息。

### `instance_query`

用途：查询指定实例详情。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例句柄；未提供时使用当前会话默认实例。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `data.type` | 连接类型。 |
| `data.state` | 当前状态。 |
| `data.config` | 当前配置，未配置时为空或部分为空。 |
| `data.created_at` | 创建时间戳。 |
| `data.connected_at` | 最近连接时间。 |
| `data.last_activity_at` | 最近收发或状态变化时间。 |
| `data.tx_bytes` | 已发送字节数。 |
| `data.rx_bytes` | 已接收字节数。 |
| `data.rx_buffer_bytes` | 普通接收缓冲区当前字节数。 |
| `data.subscriber_count` | 当前订阅数量。 |
| `data.last_error` | 最近错误摘要，可为空。 |

TCP listen 实例还必须在 `data.connection` 或等价字段中暴露：

| 字段 | 说明 |
| --- | --- |
| `data.connection.mode` | `client` 或 `server/listen`。 |
| `data.connection.listen_strategy` | 初版固定为 `single_client`。 |
| `data.connection.active_peer` | 当前活动 TCP 客户端地址；无活动连接时为空。 |
| `data.connection.connected_client_count` | 当前活动客户端数量；初版取值为 `0` 或 `1`。 |

状态要求：`Created`、`Configured`、`Connected`、`Disconnecting`、`Disconnected`、`Error`。

错误类别：`HandleNotFound`、`InvalidState`。

边界条件：

- 查询不改变实例状态。
- 已释放实例必须返回 `HandleNotFound`。
- 未提供 `handle_id` 时，必须按“实例句柄解析”规则解析默认实例。
- 缺省句柄解析可能返回 `HandleNotFound/SESSION_BINDING_MISSING` 或 `InvalidState/SESSION_ID_UNAVAILABLE`。

### `instance_use`

用途：将当前 AI 会话的默认实例切换为指定 `handle_id`。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 是 | 要绑定到当前 AI 会话的实例。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `data.bound` | 固定为 `true`。 |
| `data.previous_handle_id` | 当前会话此前绑定的句柄，可为空。 |

状态要求：`Created`、`Configured`、`Connected`、`Disconnected`、`Error`。

错误类别：`HandleNotFound`、`InvalidState`。

边界条件：

- 只影响当前 AI 会话，不影响其他会话。
- 不改变连接实例本身状态。
- 不允许绑定 `Released` 实例。

### `instance_release`

用途：释放实例并移出全局实例表。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 要释放的实例；未提供时使用当前会话默认实例。 |
| `force` | 否 | 是否强制释放连接中实例，默认 `false`。`true` 表示句柄立即失效，底层 close 转入后台，资源锁进入 `closing` tombstone。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Released`。 |
| `data.released` | 固定为 `true`。 |
| `data.close_mode` | `none`、`synchronous` 或 `background`；`force=true` 且底层资源仍打开时为 `background`。 |
| `data.resource_lock` | `released` 或 `closing_tombstone`；`closing_tombstone` 表示同一资源暂不可复用。 |
| `data.close_deadline_ms` | 后台 close 的 tombstone deadline；不适用时为空。 |

状态要求：普通释放允许 `Created`、`Configured`、`Disconnected`、`Error`；`force=true` 允许 `Created`、`Configured`、`Connected`、`Disconnecting`、`Disconnected`、`Error`。

错误类别：`HandleNotFound`、`InvalidState`、`WriteFailed`。

边界条件：

- 释放后再次操作同一 `handle_id` 必须返回 `HandleNotFound`。
- 释放必须清理会话绑定和订阅关系。
- 未提供 `handle_id` 时，必须按“实例句柄解析”规则解析默认实例。
- 连接中的实例若 `force=false`，应返回 `InvalidState` 并提示先断开。
- `Disconnecting` 中的实例若 `force=false`，应返回 `InvalidState/DISCONNECTING_IN_PROGRESS`；若 `force=true`，应立即使句柄失效，并按后台 close/tombstone 规则隔离底层资源。
- 连接中的实例若 `force=true`，返回成功时只保证句柄、会话绑定和订阅关系已失效，不保证底层 transport 已完成关闭。
- `force=true` 后同一物理资源在 `closing` tombstone 释放前再次连接，必须返回 `ResourceBusy/RESOURCE_CLOSING`，而不是复用旧资源。

## 连接配置工具

### `serial_config`

用途：为 `Serial` 实例写入或覆盖串口配置。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 是 | `Serial` 实例句柄。 |
| `port` | 是 | 串口名，例如 `COM3`。 |
| `baudrate` | 是 | 波特率，例如 `9600`、`115200`。 |
| `bytesize` | 是 | `5`、`6`、`7`、`8`。 |
| `stopbits` | 是 | `1`、`1.5`、`2`。 |
| `parity` | 是 | `None`、`Odd`、`Even`。 |
| `flow_control` | 是 | `None`、`RTS-CTS`、`XON-XOFF`。 |
| `timeout_ms` | 否 | 默认读写超时。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Configured`。 |
| `data.config` | 已保存的串口配置。 |

状态要求：`Created`、`Configured`、`Disconnected`。

错误类别：`HandleNotFound`、`InvalidState`、`InvalidArgument`。

边界条件：

- 只能配置 `Serial` 实例。
- `Connected` 或 `Disconnecting` 状态不允许覆盖配置。
- 配置阶段不占用串口；串口占用检查发生在 `port_connect`。

### `tcp_udp_config`

用途：为 `TCP` 或 `UDP` 实例写入或覆盖网络配置。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 是 | `TCP` 或 `UDP` 实例句柄。 |
| `protocol` | 是 | `TCP` 或 `UDP`，必须与实例类型一致。 |
| `ip` | 是 | 目标地址或监听地址。 |
| `port` | 是 | `1..65535`。 |
| `mode` | 是 | `client` 或 `server/listen`。 |
| `timeout_ms` | 否 | 默认连接或读写超时。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Configured`。 |
| `data.config` | 已保存的网络配置。 |

状态要求：`Created`、`Configured`、`Disconnected`。

错误类别：`HandleNotFound`、`InvalidState`、`InvalidArgument`。

边界条件：

- `protocol` 必须与实例类型一致。
- `Connected` 或 `Disconnecting` 状态不允许覆盖配置。
- 地址格式和端口范围必须在配置阶段校验。
- 监听端口冲突检查发生在 `port_connect`。

## 端口行为工具

### `port_scan`

用途：发现可用串口或探测有限范围的网络端口。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `type` | 是 | `Serial`、`TCP` 或 `UDP`。 |
| `range` | 否 | 网络扫描范围；串口扫描可忽略。初版只允许 loopback 主机和有限端口范围。 |
| `timeout_ms` | 否 | 单项探测超时。 |
| `max_results` | 否 | 最大返回数量。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `data.resources` | 可用资源或探测结果数组。 |

状态要求：无前置状态。

错误类别：`InvalidArgument`、`ConnectTimeout`、`BufferLimitExceeded`。

边界条件：

- 初版必须支持串口枚举。
- 网络扫描默认仅允许 `127.0.0.0/8` 和 `::1` 目标；非 loopback 目标必须返回 `InvalidArgument/SCAN_TARGET_NOT_ALLOWED`。非 loopback allowlist 仅属于 `02-进阶实现大纲.md` 的进阶能力，初版不得通过隐藏配置启用。
- 网络扫描必须限制单次端口数量、并发数、单项超时、总超时和返回结果数量；判定矩阵如下：

| 场景 | 错误码 |
| --- | --- |
| 非 loopback、CIDR、通配公网或私网网段、`0.0.0.0`、`::`、DNS 展开目标 | `InvalidArgument/SCAN_TARGET_NOT_ALLOWED` |
| 端口数量、端口范围或请求并发超过 `scan_max_ports`、`scan_max_concurrency` 等限制 | `BufferLimitExceeded/SCAN_RANGE_TOO_LARGE` |
| 返回结果数量或 payload 超过 `max_results` 或实现返回上限 | `BufferLimitExceeded/RESULT_TOO_LARGE` |
| 端口号、超时或数值字段越界 | `InvalidArgument/INVALID_RANGE` |
- 初版 `range` 语义为单个 loopback 主机加端口或端口区间，例如 `127.0.0.1:8000-8100`、`[::1]:9000`；不支持 CIDR、通配公网或私网网段扫描。
- `Visa` 扫描属于进阶能力，不在初版返回中出现。

### `port_connect`

用途：打开底层串口或 socket，并启动实例收发任务。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |
| `timeout_ms` | 否 | 本次连接超时；未提供时使用配置默认值。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Connected`。 |
| `data.connected_at` | 连接时间戳。 |
| `data.resource` | 已占用资源摘要。 |

状态要求：`Configured`、`Disconnected`。

错误类别：`HandleNotFound`、`InvalidState`、`ResourceBusy`、`ConnectTimeout`、`InvalidArgument`。

边界条件：

- 串口连接时必须执行物理资源独占检查。
- TCP/UDP listen 模式必须检查地址端口冲突。
- TCP listen 初版固定为 `single_client` 策略：同一实例最多维护一个活动客户端，后续连接必须拒绝或立即关闭，并通过 `instance_query` 暴露策略、活动 peer 和当前客户端数量。
- 成功连接后才能允许 `port_send`、`port_pull` 和订阅工具工作。

### `port_disconnect`

用途：断开连接、停止收发任务并释放底层资源。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |
| `timeout_ms` | 否 | 断开清理超时。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Disconnected`。 |
| `data.disconnected_at` | 断开时间戳。 |
| `data.released_resource` | 已释放资源摘要。 |

状态要求：`Connected`、`Error`；对 `Disconnected` 可按幂等成功处理。

错误类别：`HandleNotFound`、`InvalidState`、`WriteFailed`。

边界条件：

- 断开后配置和统计保留。
- 断开必须取消或停止与实例相关的收发入口。
- 当前会话绑定可以保留，但订阅应失效或标记停止。

### `port_send`

用途：向已连接实例发送一帧数据。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |
| `data` | 是 | 要发送的数据。 |
| `encoding` | 是 | `text` 或 `hex`。 |
| `text_encoding` | 否 | `encoding=text` 时使用，默认 `UTF-8`。 |
| `append_line_break` | 否 | 是否追加换行，默认 `false`。 |
| `timeout_ms` | 否 | 入队或写入超时。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Connected`。 |
| `data.queued` | 是否已进入发送队列。 |
| `data.sent_bytes` | 成功接收的待发送字节数；同步写入实现也可表示已写字节数。 |
| `data.encoding` | 实际使用编码。 |

状态要求：`Connected`。

错误类别：`HandleNotFound`、`InvalidState`、`InvalidArgument`、`WriteFailed`、`BufferLimitExceeded`。

边界条件：

- 同一实例的发送必须按调用进入队列的顺序串行处理。
- 单次 `port_send` 的数据帧不可被其他发送请求插入。
- `hex` 数据必须先通过格式校验再入队。
- `append_line_break=true` 时，文本默认追加平台无关的协议换行；具体换行内容可由后续配置文档扩展。

### `port_pull`

用途：从普通接收缓冲区拉取数据。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |
| `timeout_ms` | 否 | 等待数据的超时时间。 |
| `max_bytes` | 否 | 最多返回字节数。 |
| `encoding` | 否 | 返回数据编码，默认 `hex`。 |
| `text_encoding` | 否 | `encoding=text` 时使用。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Connected`。 |
| `data.payload` | 返回数据。 |
| `data.encoding` | 返回编码。 |
| `data.received_bytes` | 本次返回字节数。 |
| `data.remaining_rx_buffer_bytes` | 读取后普通接收缓冲区剩余字节数。 |
| `data.truncated` | 是否因 `max_bytes` 截断。 |

状态要求：`Connected`。

错误类别：`HandleNotFound`、`InvalidState`、`InvalidArgument`、`ReadTimeout`、`BufferLimitExceeded`。

边界条件：

- `port_pull` 是消耗式读取，返回的数据必须从普通 `rx` 缓冲区移除。
- `port_pull` 不影响已广播给订阅者的数据。
- `max_bytes` 必须受全局最大值限制。

### `port_clear`

用途：清空指定实例的发送队列、普通接收缓冲区或两者。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |
| `buffer_type` | 是 | `tx`、`rx` 或 `all`。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | 当前实例状态。 |
| `data.cleared` | 被清理的缓冲区类型数组。 |
| `data.dropped_tx_items` | 被丢弃的发送项数量。 |
| `data.dropped_rx_bytes` | 被丢弃的接收字节数。 |

状态要求：`Configured`、`Connected`、`Disconnected`、`Error`。

`Created` 状态没有可清理的运行时队列或缓冲区，必须返回 `InvalidState/STATE_NOT_ALLOWED`，不得作为空成功处理。

错误类别：`HandleNotFound`、`InvalidState`、`InvalidArgument`。

边界条件：

- 清空 `tx` 只影响尚未写入底层资源的发送队列项。
- 清空 `rx` 只影响普通接收缓冲区，不撤回已发送给订阅者的通知。
- `Created` 状态没有可清理缓冲区，必须返回 `InvalidState/STATE_NOT_ALLOWED`。

## 最小流式订阅工具

### `port_subscribe_stream`

用途：让当前 AI 会话订阅指定实例的接收数据广播。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |
| `trigger_condition` | 否 | 初版只支持 `on_receive`，默认 `on_receive`。 |
| `max_payload_bytes` | 否 | 单次通知最大 payload。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | `Connected`。 |
| `data.subscribed` | 固定为 `true`。 |
| `data.stream_uri` | 例如 `ports://h_ser_001/stream`。 |
| `data.trigger_condition` | `on_receive`。 |

状态要求：`Connected`。

错误类别：`HandleNotFound`、`InvalidState`、`InvalidArgument`、`BufferLimitExceeded`。

边界条件：

- 初版只支持 `on_receive`。
- 同一 AI 会话重复订阅同一实例应幂等成功，或返回当前订阅状态。
- 多个 AI 会话订阅同一实例时，应各自收到同一份接收数据通知。
- 订阅不消费普通 `rx` 缓冲区。
- 通知投递必须遵守 [06-并发与缓冲区设计](06-并发与缓冲区设计.md) 和 [11-配置与运行说明](11-配置与运行说明.md) 定义的单订阅者、单实例、全局限速和 burst 策略；超限时合并或丢弃通知并累计 `dropped_notifications`，不得阻塞接收任务或普通 `rx`。

### `port_unsubscribe_stream`

用途：取消当前 AI 会话对指定实例的接收流订阅。

入参：

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| `handle_id` | 否 | 目标实例；未提供时使用当前会话默认实例。 |

成功返回：

| 字段 | 说明 |
| --- | --- |
| `state` | 当前实例状态。 |
| `data.subscribed` | 固定为 `false`。 |
| `data.stream_uri` | 被取消的 stream URI。 |

状态要求：`Connected`、`Disconnected`、`Error`。

错误类别：`HandleNotFound`、`InvalidState`。

边界条件：

- 只取消当前 AI 会话的订阅，不影响其他会话。
- 对未订阅会话调用可按幂等成功处理。
- 断开连接后订阅应停止；是否需要显式取消由 `05` 定义。

## 数据编码契约

### `text`

- `data` 按 `text_encoding` 转为字节。
- 默认 `text_encoding` 为 `UTF-8`。
- 文本编码失败返回 `InvalidArgument`。
- `append_line_break=true` 只适用于 `encoding=text`；如实现允许 Hex 追加换行，必须在返回中标明追加的字节。

### `hex`

- 支持空格分隔格式，例如 `41 42 43 0D 0A`。
- 支持连续格式，例如 `4142430D0A`。
- 字符数量必须为偶数，且只能包含十六进制字符和允许的分隔空白。
- 非法 Hex 返回 `InvalidArgument`。
- 返回二进制数据时默认使用大写空格分隔 Hex，便于 AI 阅读。
