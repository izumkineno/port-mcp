# 08 端口 MCP 测试与验收计划

本文档承接 [01-初版工程功能设计大纲](01-初版工程功能设计大纲.md)、[03-初版工具契约](03-初版工具契约.md)、[04-运行时架构设计](04-运行时架构设计.md)、[05-状态机与资源生命周期](05-状态机与资源生命周期.md)、[06-并发与缓冲区设计](06-并发与缓冲区设计.md) 与 [07-错误模型与返回格式](07-错误模型与返回格式.md)，把初版工程设计转化为可执行、可回归、可手工验收的测试矩阵。

08 的主体只覆盖初版必测项。进阶能力来自 [02-进阶实现大纲](02-进阶实现大纲.md)，本文只保留测试占位，不把 VISA、协议 helper、高级订阅、链路维持或异常注入列为初版编码前硬门槛。

## 设计结论

- CI 不依赖真实硬件，默认只运行单元测试、MockTransport 集成测试、TCP/UDP loopback 测试、并发测试和错误模型测试。
- 串口能力进入手工验收，优先在 Windows 上使用真实串口设备、USB-TTL 回环或虚拟串口对验证。
- Windows 是初版必测平台；Linux 和 macOS 作为可选抽测平台，主要验证串口命名、权限和换行差异。
- 所有工具必须覆盖成功路径、状态拒绝、参数错误、句柄错误和关键资源错误。
- 每条会改变状态、资源锁、后台任务、缓冲区或订阅关系的路径，都必须能通过测试观察到结果。
- 错误模型测试不仅检查 `category`，还要检查 `code`、`message`、`recovery_hint`、`details`、`retryable` 和状态影响。
- 验收报告应区分自动化通过、手工通过、可选抽测、已知限制和阻塞问题。

## 测试分层

| 层级 | 运行环境 | 是否初版必需 | 覆盖重点 |
| --- | --- | --- | --- |
| 单元测试 | CI、本地 | 必需 | 参数校验、状态矩阵、错误码映射、缓冲区、资源 key、脱敏规则。 |
| MockTransport 集成测试 | CI、本地 | 必需 | 生命周期、发送队列、接收缓冲、订阅、后台任务失败、断开释放。 |
| TCP loopback 测试 | CI、本地 | 必需 | 真实异步 I/O、连接、收发、对端关闭、地址冲突、超时。 |
| UDP loopback 测试 | CI、本地 | 必需 | UDP bind、基础 datagram 收发、地址端口冲突、释放后重绑。 |
| 并发与多会话测试 | CI、本地 | 必需 | FIFO、队列满拒绝、共享 rx、订阅隔离、会话绑定隔离。 |
| 错误模型测试 | CI、本地 | 必需 | 统一返回、错误类别、细分错误码、恢复建议、脱敏、状态影响。 |
| 串口手工验收 | Windows 本地 | 必需 | 枚举、连接、串口回环、一问一答、独占、权限或设备错误。 |
| Linux/macOS 抽测 | 本地 | 可选 | 串口路径、权限、换行、TCP/UDP loopback。 |

## 自动化测试范围

自动化测试的目标是让没有硬件的开发者也能稳定验证初版核心行为。

| 测试组 | 主要用例 | 通过标准 |
| --- | --- | --- |
| `unit_arguments` | 必填字段、枚举、数值范围、地址格式、Hex 和 text 编码。 | 非法输入返回 `InvalidArgument`，`code` 和 `details.field` 可定位问题。 |
| `unit_state_matrix` | 所有工具在 `Created`、`Configured`、`Connected`、`Disconnecting`、`Disconnected`、`Error`、`Released` 下的允许性。 | 允许、幂等成功、拒绝结果与 [05-状态机与资源生命周期](05-状态机与资源生命周期.md) 一致。 |
| `unit_error_mapping` | 03 中每类错误和 07 中主要 code 的映射。 | 每个失败返回 `ok=false`，保留稳定 `category` 和细分 `code`。 |
| `unit_redaction` | 路径、用户名、环境变量、payload、OS 错误文本和堆栈。 | 工具返回只含脱敏摘要，日志可标记 sensitive。 |
| `unit_buffers` | tx 队列容量、rx 丢旧保新、pull 截断、订阅 payload 截断。 | 统计字段、`truncated`、`dropped_*` 与 [06-并发与缓冲区设计](06-并发与缓冲区设计.md) 一致。 |
| `unit_runtime_budget` | `buffer_bytes_budget`、`queued_bytes_budget` reserve/归还、绝对硬上限配置校验和长时间运行。 | reserve 失败行为、丢弃后重试、`port_clear`/disconnect/release 归还、`ABS_MAX_TOTAL_BUFFER_BYTES` 和 `ABS_MAX_TOTAL_QUEUED_BYTES` 越界拒绝、预算不泄漏均符合 06、11。 |
| `unit_notification_rate` | 单订阅者、单实例和全局通知限速、burst、合并/丢弃策略和限速日志。 | 高频输入不会产生无界通知风暴；`dropped_notifications`、`coalesced`、限速日志和未超限订阅者投递均符合 06。 |
| `unit_resource_keys` | serial、tcp-listen、udp-bind 资源 key 规范化。 | 相同资源生成相同 key，不同资源不会误冲突。 |

## MockTransport 集成测试

MockTransport 是初版自动化测试的一等测试基座，但不作为用户可见工具类型暴露。

| 用例 | 步骤 | 断言 |
| --- | --- | --- |
| 基础生命周期 | create -> config -> connect -> send -> pull -> disconnect -> release。 | 状态按 `Created` 到 `Released` 迁移，释放后同句柄返回 `HandleNotFound`。 |
| 接收缓冲消费 | Mock 注入两段接收数据，连续 `port_pull`。 | 第一次消费指定字节，第二次只返回剩余数据。 |
| 发送 FIFO | 多个会话并发 `port_send`。 | Mock 观察到的帧顺序等于成功入队顺序，单帧不交织。 |
| 发送队列满 | 将 tx 队列填满后继续发送。 | 新发送返回 `BufferLimitExceeded/TX_QUEUE_FULL`，已排队项不丢。 |
| rx 丢旧保新 | 注入超过 `rx_buffer_max_bytes` 的数据。 | 保留最新数据，累计 `rx_dropped_bytes`，工具不因此失败。 |
| 订阅广播 | 两个会话订阅同一实例后注入数据。 | 两个订阅者均收到通知，普通 `rx` 仍可被 `port_pull` 消费。 |
| 慢订阅者 | 一个订阅者队列满，另一个正常消费。 | 慢订阅者只丢自己的旧通知，不阻塞接收任务或其他订阅者。 |
| 后台读失败 | Mock 接收任务返回致命错误。 | 实例进入 `Error`，`last_error` 可查询，收发工具被拒绝。 |
| 断开清理 | connected 状态调用 `port_disconnect`。 | 发送入口关闭，后台任务退出，订阅停止，资源锁释放。 |
| 强制释放 | connected 状态调用 `instance_release(force=true)`。 | 句柄立即失效，资源锁进入 `closing` tombstone，同资源在后台 close 完成或超时前返回 `RESOURCE_CLOSING`。 |

## TCP/UDP Loopback 测试

Loopback 测试验证真实异步 I/O 路径，不依赖外部网络。

### TCP

| 用例 | 断言 |
| --- | --- |
| TCP client 连接本地 echo server | `port_connect` 成功，`port_send` 后可 `port_pull` 到回显。 |
| TCP 对端主动关闭 | 后台任务观察到关闭，实例进入 `Error` 或断开路径定义的状态，并记录 `last_error`。 |
| TCP 连接超时 | 返回 `ConnectTimeout/CONNECT_TIMEOUT`，状态进入 `Error`。 |
| TCP listen 地址冲突 | 第二个相同 listen 地址返回 `ResourceBusy/TCP_LISTEN_ADDR_BUSY`。 |
| 释放后重连 | `port_disconnect` 或 `instance_release` 后相同地址可再次使用。 |

### UDP

| 用例 | 断言 |
| --- | --- |
| UDP bind 后发送到本地 echo 端 | 可以收到 datagram，payload 与发送内容一致。 |
| UDP 连续 datagram 字节流语义 | 连续发送两个 datagram 后，`port_pull` 按接收顺序返回串接 payload，不承诺 datagram 边界。 |
| UDP 订阅 datagram 元数据 | 订阅通知按单个 datagram 触发时标记 `transport_protocol=udp` 和 `datagram=true`。 |
| UDP bind 地址冲突 | 第二个相同 bind 地址返回 `ResourceBusy/UDP_BIND_ADDR_BUSY`。 |
| UDP 释放后重绑 | 断开或释放后同地址端口可重新绑定。 |
| UDP 空读超时 | 返回 `ReadTimeout/READ_TIMEOUT`，实例状态不变。 |

## 状态机验收矩阵

状态机测试必须覆盖每个工具的允许状态和拒绝状态。建议把 [05-状态机与资源生命周期](05-状态机与资源生命周期.md) 的工具允许状态矩阵转为数据驱动测试。

| 路径 | 断言 |
| --- | --- |
| create 成功 | 新实例进入 `Created`，可被 `instance_list` 和 `instance_query` 看到。 |
| config 成功 | 对应实例进入或保持 `Configured`。 |
| connect 成功 | 进入 `Connected`，后台发送/接收任务启动，资源锁已占用。 |
| connect 失败 | 进入 `Error`，写入 `last_error`，资源锁按 05 规则回滚或释放。 |
| disconnect 成功 | 进入 `Disconnected`，后台任务停止，资源锁释放。 |
| disconnected 再 disconnect | 幂等成功，不产生新错误。 |
| disconnect 失败 | 进入或保持 `Error`，可查询和释放。 |
| release 成功 | 实例从表移除，同句柄后续操作返回 `HandleNotFound`。 |
| Error 恢复 | `instance_query`、`port_disconnect`、`port_clear`、`instance_release` 可按 05 规则工作。 |
| ReadTimeout | `port_pull` 返回 `ReadTimeout`，状态保持 `Connected`。 |

## 工具契约验收矩阵

每个初版工具至少要有一个成功用例、一个状态拒绝用例、一个参数或句柄负例。无业务错误的工具也要覆盖返回外形。

| 工具 | 必测成功路径 | 必测失败路径 |
| --- | --- | --- |
| `instance_create` | 创建 serial、tcp、udp 三类实例。 | 非法实例类型返回 `InvalidArgument`。 |
| `instance_list` | 返回当前实例摘要。 | 无业务错误；内部错误按统一失败外形包装。 |
| `instance_query` | 查询状态、统计、`last_error`；省略 `handle_id` 时使用当前会话默认实例。 | 不存在句柄返回 `HandleNotFound`；未绑定默认实例返回 `SESSION_BINDING_MISSING`。 |
| `instance_use` | 当前会话绑定默认实例。 | `Released` 或不存在句柄返回 `HandleNotFound`；缺失稳定 session_id 返回 `SESSION_ID_UNAVAILABLE`。 |
| `instance_release` | disconnected/error 实例释放。 | connected 普通释放返回 `InvalidState`，force 成功。 |
| `serial_config` | serial 实例配置成功。 | 非 serial 实例、非法 baudrate、错误状态。 |
| `tcp_udp_config` | tcp/udp 实例配置成功。 | 非法地址、非法端口、错误实例类型。 |
| `port_scan` | 扫描返回候选端口或空列表。 | 非法扫描范围、扫描超时、并发超限、范围过大或结果过大。 |
| `port_scan` loopback 限制 | loopback 端口区间可扫描。 | 非 loopback、CIDR、通配目标、`0.0.0.0`、`::` 或 DNS 展开目标返回 `SCAN_TARGET_NOT_ALLOWED`；端口数或并发超限返回 `SCAN_RANGE_TOO_LARGE`；结果或 payload 超限返回 `RESULT_TOO_LARGE`。 |
| `port_connect` | 配置后连接成功。 | 资源冲突、超时、未配置、错误状态。 |
| `port_disconnect` | connected 断开成功，disconnected 幂等成功。 | 不存在句柄、错误状态、断开失败。 |
| `port_send` | text 和 hex 入队成功。 | 非法 hex、编码失败、队列满、单帧过大。 |
| `port_pull` | 消耗 rx 并返回数据。 | 超时、`max_bytes` 超限、错误状态。 |
| `port_clear` | 清理 tx、rx、all；省略 `handle_id` 时使用当前会话默认实例。 | 非法 `buffer_type`、`Created` 等错误状态、缺失默认实例。 |
| `port_subscribe_stream` | 当前会话订阅 on_receive。 | 非法 trigger、payload 上限超限、订阅者数量超过 `max_subscribers_per_instance` 返回 `SUBSCRIBER_LIMIT_EXCEEDED`、错误状态。 |
| `port_unsubscribe_stream` | 当前会话取消订阅，未订阅可幂等成功。 | 不存在句柄、错误状态。 |

## 错误模型验收

错误模型测试按 [07-错误模型与返回格式](07-错误模型与返回格式.md) 执行，重点是返回结构稳定且便于 AI 自查。

| 类别 | 至少覆盖的 code | 状态影响断言 |
| --- | --- | --- |
| `HandleNotFound` | `HANDLE_NOT_FOUND`、`HANDLE_RELEASED`、`SESSION_BINDING_MISSING` | 无实例可变更。 |
| `InvalidState` | `STATE_NOT_ALLOWED`、`CONNECTED_RELEASE_REQUIRES_FORCE`、`SESSION_ID_UNAVAILABLE` | 不改变状态，除非 05 明确状态转移。 |
| `InvalidArgument` | `MISSING_REQUIRED_FIELD`、`INVALID_ENUM_VALUE`、`INVALID_RANGE`、`SCAN_TARGET_NOT_ALLOWED`、`INVALID_HEX`、`TEXT_ENCODING_FAILED` | 不改变状态。 |
| `ResourceBusy` | `SERIAL_PORT_BUSY`、`TCP_LISTEN_ADDR_BUSY`、`UDP_BIND_ADDR_BUSY`、`RESOURCE_CLOSING`、`RESOURCE_LOCK_STALE` | `port_connect` 失败进入 `Error` 或按 05 的 tombstone/stale-lock 规则处理。 |
| `ConnectTimeout` | `CONNECT_TIMEOUT`、`SERIAL_OPEN_TIMEOUT`、`SCAN_TIMEOUT` | 连接类失败进入 `Error`，扫描超时不污染实例。 |
| `ReadTimeout` | `READ_TIMEOUT`、`NO_DATA_AVAILABLE` | 不改变状态。 |
| `WriteFailed` | `WRITE_IO_FAILED`、`TRANSPORT_CLOSED`、`TASK_FAILED`、`DISCONNECT_FAILED` | 致命读写或断开失败进入或保持 `Error`。 |
| `BufferLimitExceeded` | `TX_QUEUE_FULL`、`TX_FRAME_TOO_LARGE`、`PULL_MAX_BYTES_EXCEEDED`、`SUBSCRIBER_PAYLOAD_TOO_LARGE`、`SUBSCRIBER_LIMIT_EXCEEDED`、`SCAN_RANGE_TOO_LARGE`、`RESULT_TOO_LARGE` | 不改变状态。 |

每个失败断言至少包含：

- `ok=false`。
- `tool` 与当前工具一致。
- 请求中有 `handle_id` 时原样返回。
- `error.category` 和 `error.code` 稳定。
- `error.message` 可读。
- `error.recovery_hint` 指向下一步动作。
- `error.details` 不包含未脱敏敏感信息。

## 并发与多会话验收

| 场景 | 断言 |
| --- | --- |
| 同实例并发发送 | 成功入队顺序决定底层写入顺序，单帧不交织。 |
| 不同实例并发发送 | 互不阻塞，统计和缓冲区不串线。 |
| 发送中断开 | 新发送被拒绝，已开始写入帧按实现语义完成或失败，但状态和错误可解释。 |
| 发送队列满 | 新 `port_send` 返回 `TX_QUEUE_FULL`，已排队项保留。 |
| 多会话默认实例 | `instance_use` 绑定按 `session_id` 隔离。 |
| 缺省句柄解析 | 工具未传 `handle_id` 且允许缺省时，使用当前会话默认实例；成功返回必须包含解析后的 `handle_id`。 |
| 缺失稳定 session_id | 默认实例解析、`instance_use`、订阅和取消订阅返回 `SESSION_ID_UNAVAILABLE`，显式 `handle_id` 且不依赖当前会话身份的工具仍可使用。 |
| 单会话开发模式 | 配置 `session_mode=single_dev` 只能显式本地启用；启用后工具返回和日志必须标记 `session_mode=single`；远程或多客户端配置下必须拒绝启用。 |
| 多会话共享 rx | 任一会话 `port_pull` 消费同一个普通 `rx` 缓冲区。 |
| 多会话订阅 | 每个订阅者有独立队列，一个会话取消不影响其他会话。 |
| 慢订阅者 | 只丢慢订阅者自己的旧通知，并累计 dropped 统计。 |
| 通知限速 | 高频接收输入触发单订阅者、单实例或全局限速时，只合并或丢弃通知并累计 dropped 统计，不阻塞接收任务、其他订阅者或普通 `rx`。 |
| clear 与收发并发 | `port_clear` 只影响目标缓冲区，不撤回正在写入帧或已广播通知。 |
| release 与并发操作 | release 后句柄立即不可用；强制释放时资源锁进入 `closing` tombstone，后台清理不再通过旧句柄返回。 |

## 资源锁验收

| 资源 | 锁 key | 必测行为 |
| --- | --- | --- |
| Serial | `serial:<port>` | 同一串口同一时刻只能有一个 connected 实例；强制释放后 close 完成前同资源返回 `RESOURCE_CLOSING`。 |
| TCP listen | `tcp-listen:<ip>:<port>` | 相同监听地址端口冲突；强制释放后 close 完成前同地址返回 `RESOURCE_CLOSING`。 |
| UDP bind | `udp-bind:<ip>:<port>` | 相同绑定地址端口冲突；强制释放后 close 完成前同地址返回 `RESOURCE_CLOSING`。 |

资源锁测试必须覆盖：申请成功、冲突失败、连接失败回滚、断开释放、普通释放、强制释放进入 `closing` tombstone、后台 close 成功释放 tombstone、后台 close 超时留下 `background_close_error` 和 `stale_close=true` 诊断事件、stale tombstone 释放后底层仍 busy 时返回 `RESOURCE_LOCK_STALE` 或具体 busy code。

`force_close_deadline_ms` 配置测试必须覆盖低于最小值、超过最大值、`0` 和负数被拒绝；deadline 到达后同资源重试路径返回 `RESOURCE_LOCK_STALE` 或具体 busy code，并记录限速后的 stale/backoff 诊断。

## 串口手工验收

串口不作为 CI 必需项，但初版在发布或进入真实设备调试前必须完成 Windows 手工验收。

| 项目 | 最低要求 | 通过标准 |
| --- | --- | --- |
| 设备准备 | 真实串口设备、USB-TTL 回环或虚拟串口对三选一。 | 能被系统枚举，并能形成可预测收发路径。 |
| 串口枚举 | 调用扫描或配置前辅助流程。 | 能看到目标串口名称，例如 `COM3`。 |
| 串口连接 | `serial_config` 后 `port_connect`。 | 状态进入 `Connected`，资源锁占用。 |
| 串口收发 | 发送 text 和 hex，使用回环或设备响应读取。 | `port_send` 成功，`port_pull` 收到预期字节。 |
| 串口独占 | 同一串口创建第二个连接。 | 返回 `ResourceBusy/SERIAL_PORT_BUSY`。 |
| 错误路径 | 不存在串口、权限不足或被其他程序占用。 | 返回结构化错误，不泄漏敏感路径或堆栈。 |
| 断开释放 | `port_disconnect` 和 `instance_release`。 | 资源释放后可重新连接。 |

手工验收记录至少包含：平台版本、设备或虚拟串口方案、串口名、测试工具调用摘要、失败截图或日志摘要、最终通过/不通过结论。

## 平台验收口径

| 平台 | 初版门槛 | 说明 |
| --- | --- | --- |
| Windows | 必测 | CI 或本地自动化通过，串口手工验收通过。 |
| Linux | 可选抽测 | 建议验证 TCP/UDP loopback、串口路径和权限错误。 |

如果 Linux 或 macOS 抽测失败，不阻塞初版 Windows 验收，但必须记录为已知限制或后续修复项。

## 非初版进阶测试占位
